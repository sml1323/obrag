Obsidian RAG — Document Sync & RAG Query Pipeline
📥 Document Sync Flow
Vault 변경 감지 → 증분 동기화
1
📁 Obsidian Vault 스캔
FolderScanner가 Vault 경로의 .md 파일 목록 스캔
2
🔍 변경 파일 감지
FileTracker가 파일 해시(SHA-256) 비교로 추가/수정/삭제 분류
added · modified · deleted · unchanged
3
✂️ Semantic Chunking
마크다운 구조 인식 청킹. 2-Pass 전처리로 코드 블록 내 # 보호
min_chunk: 200 · max_chunk: 1500
4
🧮 벡터 임베딩
EmbeddingStrategy로 텍스트 → 벡터 변환
OpenAI | E5 | SentenceTransformer | Ollama
5
💾 ChromaDB Upsert
변경분만 Upsert, 삭제 파일은 ID Diff로 자동 정리
upsert() · delete() · SyncRegistry 기록
✅ 해시 기반 증분 동기화로 전체 재처리 없이 변경분만 처리
💬 RAG Query Flow
사용자 질문 → AI 응답 생성
1
💭 사용자 질문 입력
Next.js 프론트엔드에서 REST API로 전달
2
🔎 벡터 유사도 검색
Retriever가 ChromaDB에서 Top-K 유사 청크 검색
L2 Distance → Score: 1/(1+d)
3
🎯 LLM Reranking
검색 결과를 LLM으로 재평가하여 관련성 높은 문서 선별
Relevance Scoring · Context Filtering
4
📝 프롬프트 조립
PromptBuilder가 컨텍스트 + 이력 + 질문을 템플릿에 주입
System · Context · History · User
5
🤖 LLM 응답 생성
LLMStrategy Protocol로 런타임 모델 선택
OpenAI | Gemini | Ollama
6
📡 SSE 스트리밍 응답
Server-Sent Events로 실시간 토큰 단위 전송
stream_generate() → EventStream
⚡ 검색 → 재랭킹 → 생성의 3단계 파이프라인, 실시간 스트리밍
