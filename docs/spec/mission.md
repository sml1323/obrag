# Product Mission

## Vision

**Obsidian AI**는 사용자의 옵시디언 노트를 AI가 학습하여 질문에 답하고, PARA 방법론에 기반한 프로젝트 진행률 및 에빙하우스 망각곡선 기반 복습 주기를 관리하는 **통합 지식 관리 도구**입니다.

> "제2의 뇌를 진정한 지식 파트너로 만들다"

---

## Core Philosophy (개발 철학)

- **Unit Testing**: 모든 핵심 기능(임베딩, 동기화, 복습 로직 등)은 단위 테스트를 통해 검증 가능해야 함.
- **Debuggable**: 사용자가 디버거(Pdb 등)를 통해 ETL 파이프라인과 RAG 로직의 작동 원리를 투명하게 확인할 수 있어야 함.
- **Granular Implementation**: 블록/함수 단위의 작은 구현을 지향하며, 각 모듈은 독립적으로 테스트 및 교체 가능해야 함.
- **Incremental Sync**: 전체 재처리 대신 변경된 파일만 업데이트하여 효율성 확보.

---

## Key Goals

### 1. 지능형 RAG 채팅 (Semantic Search & Chat)
- Obsidian `.md` 파일들을 벡터화하여 의미 기반 검색 제공
- 멀티 모델 지원 (OpenAI, Gemini, Ollama)
- 증분 업데이트로 효율적인 Vector DB 관리

### 2. PARA 프로젝트 대시보드
- `Project/` 폴더 기반 프로젝트 자동 인식
- 진척도 관리 및 유기(Stale) 프로젝트 식별
- 시각화된 카드형/리스트형 대시보드

### 3. 에빙하우스 복습 엔진 (Memory Boost)
- `#review` 태그 기반 복습 대상 관리
- 에빙하우스 주기(1일, 3일, 7일, 14일, 30일)에 따른 알림
- 랜덤 복습 기능으로 전체 지식 훑기

---

## Target Users

- 옵시디언을 **제2의 뇌**로 활용하는 사용자
- 지식의 유기적 연결과 체계적인 프로젝트 관리를 원하는 사용자
- 학습한 내용을 장기 기억으로 정착시키고 싶은 사용자
